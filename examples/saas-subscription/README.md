# SaaS Subscription Platform — neutrino-schema demo

Seven PostgreSQL tables → fully typed Rust types in one command.

## Before: 200 lines of SQL

```sql
CREATE TABLE users (
    id            BIGSERIAL    PRIMARY KEY,
    email         VARCHAR(255) NOT NULL UNIQUE,
    name          VARCHAR(255) NOT NULL,
    avatar_url    TEXT,
    is_active     BOOLEAN      NOT NULL DEFAULT true,
    metadata      JSONB,
    created_at    TIMESTAMPTZ  NOT NULL DEFAULT now(),
    updated_at    TIMESTAMPTZ  NOT NULL DEFAULT now()
);

CREATE TABLE teams (
    id            BIGSERIAL    PRIMARY KEY,
    owner_id      BIGINT       NOT NULL REFERENCES users(id),
    ...
);

CREATE TABLE subscriptions (
    id                    BIGSERIAL    PRIMARY KEY,
    team_id               BIGINT       NOT NULL REFERENCES teams(id),
    plan_id               BIGINT       NOT NULL REFERENCES plans(id),
    ...
);
-- 4 more tables …
```

## Quick start

### 1. Create the database

```bash
createdb saas_demo
psql -d saas_demo -f schema.sql
```

### 2. Bootstrap a config file

```bash
neutrino-schema init --database-url "postgres://localhost/saas_demo"
```

This creates `neutrino-schema.toml` in the current directory:

```toml
[databases.default]
url = "postgres://localhost/saas_demo"
provider = "postgres"

[generator]
output = "src/types"
module_name = "types"
render_mode = "clean"
```

### 3. Generate types

```bash
neutrino-schema generate
```

### 4. Use them in your project

```toml
# Cargo.toml
[dependencies]
neutrino-schema = "0.6"
chrono = { version = "0.4", features = ["serde"] }
serde_json = "1"
```

## Without a live database

Export the schema to JSON, then generate code offline:

```bash
# Export (requires a live database)
neutrino-schema export --database-url "postgres://localhost/saas_demo" --pretty -o saas_demo.schema.json

# Generate (no database needed)
neutrino-schema generate --from-ir saas_demo.schema.json --output src/types
```

This is useful for CI pipelines, code review, and generating types when the database isn't running.

## Supported databases

The same schema works with PostgreSQL, MySQL/MariaDB, or SQLite.
Just change the `--database-url`:

```bash
# PostgreSQL
neutrino-schema generate --database-url "postgres://localhost/saas_demo" --output src/types

# MySQL
neutrino-schema generate --database-url "mysql://root:pass@localhost/saas_demo" --output src/types

# SQLite
neutrino-schema generate --database-url "sqlite:./saas_demo.db" --output src/types
```

## Validation

`neutrino-schema` validates your schema before code generation:

```bash
# Validation runs automatically on generate and export
neutrino-schema generate --from-ir saas_demo.schema.json --output src/types
# → prints warnings for orphan enums, errors for duplicate tables, etc.
```

Or use it programmatically:

```rust
use neutrino_schema::{validate, ValidationLevel};

let report = validate(&schema);
if report.has_errors() {
    for entry in report.errors() {
        eprintln!("[error] {}: {}", entry.location.as_deref().unwrap_or("(global)"), entry.message);
    }
}
```

## What `neutrino-schema` handled automatically

- **7 tables → 7 Rust files + mod.rs** — zero manual typing
- **Foreign keys → `i64` relation fields** — no manual mapping
- **Nullable columns → `Option<T>`** — nullability encoded in the type system
- **Timestamp columns → `chrono::DateTime<chrono::Utc>`** — idiomatic Rust types
- **JSONB columns → `serde_json::Value`** — zero-config JSON support
- **BOOLEAN → `bool`** — not integer, not magic strings
- **Primary key metadata** — `USERS_PRIMARY_KEY` constants for runtime use
- **`_id` suffix heuristic** — relationship inference without FK queries
